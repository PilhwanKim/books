# 11장 입력 유효성 검사 및 테스트 구조화

지금까지는 한개의 `functional_tests.py`, `tests.py` 로 태스트를 작성했다.

이후로 새로운 기능을 만들면 테스트가 늘어날 텐데 한 파일에 너무 많은 테스트가 있어서 찾기 힘들어진다.

그래서 테스트 코드를 여러개의 파일로 쪼개는 시간을 가진다.
